```python
def open_new_window(self, type_hint: str = "window") -> None:
    """Open a new browser window or tab and switch focus to it."""
    self._driver.switch_to.new_window(type_hint)

def switch_to_latest_window(self, timeout: int = 10) -> None:
    """Wait for a new window to appear and switch focus to it."""
    from selenium.webdriver.support import expected_conditions as EC
    old_handles = set(self._driver.window_handles)
    self._wm.wait_for(EC.new_window_is_opened(list(old_handles)), "new window to appear", timeout=timeout)
    new_handles = set(self._driver.window_handles) - old_handles
    new_handle = new_handles.pop()
    self._driver.switch_to.window(new_handle)
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Workflow JSON → `ElementActions.execute` dispatch | Untrusted JSON's `action` string crosses into Python control flow via Pydantic-validated `ActionType` enum. New window-switch branches are only reachable if Pydantic accepted the value. |
| `ElementActions` → `BasePage` window methods | Internal trust boundary. Window-switch branches pass no user-supplied string to `BasePage` — the `type_hint` values ("window", "tab") are hardcoded string literals in the dispatch branch, not derived from JSON input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | New `SWITCH_TO_*` dispatch branches | mitigate | Branches reached only after Pydantic enum validation of the JSON `action` field. The `else: raise ElementActionError` fallback (existing) still catches any unhandled `ActionType`, so adding branches without tests will fail loudly. Acceptance criteria require the `else` to remain LAST and present. |
| T-03-06 | Elevation of Privilege | `type_hint` in `open_new_window` dispatch | mitigate | `type_hint` values are hardcoded string literals (`"window"`, `"tab"`) in the dispatch branch — not derived from `element.value` or any user-supplied JSON field. No injection vector. Acceptance criteria enforce the exact literal strings. |
| T-03-07 | Denial of Service | `SWITCH_TO_LATEST_WINDOW` dispatch hanging | accept | Dispatch delegates to `BasePage.switch_to_latest_window()` which uses `WaitManager` with a 10-second timeout (mitigated at the BasePage layer, verified in T-03-02). No additional timeout needed at the dispatch layer. |
| T-03-08 | Repudiation | JSON fixture `new_window_tab.json` in testdata | accept | Test fixture only. No production execution path touches `testdata/` at runtime. File is used only by loader verification and future smoke tests. |
